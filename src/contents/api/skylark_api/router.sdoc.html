router

This module contains some types and methods related to routing.

.baseUrl()

router.baseUrl(path)

Get or set the base url for the router.

Return value

the base url value on getting or the router module object for chains call on setting.

Parameter

  • path
    TYPE : String
    The base url to be setted

.current()

router.current()

Get the current route path.

Return value

a plain object contains the following properties :

Parameter

  • route
    TYPE:Route
    a Route object with route define such as /section/:id, /home
  • params
    TYPE:PlainObject
    a plain object contains param-value pairs for parameterized route, such as {id :3} for the route /section/:id

.on()

router.on(names, data, callback, ctx)

Register a global router event listener.

The router supports global events and you can implement hooker functions by listening to global events.
Built-in global events have routing before current route change and routed after current route change.

Return value

the router module object for chains call.

Parameter

  • names
    TYPE: String
    One or more space-separated event types
  • data
    TYPE: PlainObject Optional
    Data to be passed to the handler in event.data when an event is triggered.
  • callback
    TYPE: Function
    the callback function
  • context
    TYPE: Object OPtional
    the context object of the callback function

.Route

router.Route

A skylark class type contains a route define, inherited from langx.Evented.

The Route Object contains four build-in events, When the route is changed, these events are triggered in order.

  1. exiting (for previous route)
  2. entering (for new route)
  3. entered (for new route)
  4. exited (for previous route)
